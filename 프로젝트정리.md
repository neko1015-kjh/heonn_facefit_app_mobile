## 1. 한눈에 보기

**HeOnn FaceFit** = AI 얼굴 분석 + **온열 유기(방짜유기) 괄사(가르마) 뷰티 디바이스** 동반 앱.
- 사용자가 얼굴을 촬영 → AI가 **비대칭·부기·다크서클·주름·피부톤·나이** 등을 분석
- 분석 결과에 맞춰 **케어 가이드·맞춤 제품**을 추천하고, **Before/After 변화**를 추적
- 모바일 앱(안드로이드) + 백엔드(AI 서버) + 웹 미리보기 + 운영 대시보드로 구성

**개발 방식**: "바이브 코딩" — 사용자(비개발자)가 자연어로 요청하면 Claude가 구현·배포.

5. **"대시보드"는 항상 두 곳을 의미** → ① 공개 HF 대시보드 ② claude.ai 아티팩트. **둘 다 똑같이 갱신.**

## 7. AI 엔진 — 분석 파이프라인 & 보정 (★핵심)

사진 1장 → `_detect_and_score`:
1. **다운스케일**(긴 변 1280px) — 속도(231→131ms)·해상도 일관성
2. **MediaPipe 478점 검출**(공유 검출기 + lock, 분석은 스레드 오프로드 → 동시 접속 안 멈춤)
3. **정면 게이팅**: 머리 각도(yaw/pitch)가 크면 재촬영 안내 (1단계 보정)
4. **각도 보정(roll)**: 얼굴을 똑바로 세운 좌표에서 점수 계산 (2단계) — 기울여 찍어도 같은 점수
5. **조명 보정**: 그레이월드+밝기정규화 후 피부톤 측정 (3단계) — 조명 영향 제거
6. **점수 산출(근거화)**: 측정값(%·배)을 **정상범위 기준점(anchor)** 으로 0~100 환산 + **근거값**("좌우 폭 차이 5.9%")
   - 비대칭 / 부기 / 다크서클 / 주름 / (피부톤 brightness·redness)
7. **동일인 판별**: **SFace 얼굴 임베딩(128차원) 코사인 유사도 ≥ 0.363**이면 같은 사람. 직전 사진과 비교(임베딩 없으면 즉석 재계산, 비교 불가 시 차단 안 함). 각도·표정에 강함.
8. 성별·나이(Caffe), 3D 표시용 좌표(각도 보정 반영) 반환

**검증 체계(페이지)**: `/stability`(반복촬영 안정성), `/accuracy`(거울대칭=정답100 기준 정확도), `/zone`(한쪽 부풀려 케어부위 정확도).

## 9. 배포 방법 (명령어)

**백엔드/대시보드 배포(무료·즉시):**
```
cd ../heonn_facefit_app_backend
git add <파일> && git commit -m "..."
git push origin master                                              # GitHub
git push https://huggingface.co/spaces/neko1015/facefit-backend master:main   # HF 자동 재빌드
```
- 푸시 후 HF가 ~10~60초 재빌드. `/health`로 확인.
- 대시보드는 dashboard.html(HF) + 아티팩트(Artifact 도구, 같은 URL로 redeploy) **둘 다** 갱신.

**앱 APK 빌드(사용자가 "빌드해" 할 때만):**
```
cd heonn_facefit_app_mobile
EXPO_TOKEN="<토큰>" npx eas-cli build -p android --profile preview --non-interactive --no-wait
```
- 완료되면 APK 다운로드 링크 생성 → 폰에서 열어 설치.
- **무료 대안**: 공개 GitHub 저장소라 GitHub Actions로 Gradle 빌드도 가능(무제한). 또는 로컬 `npm start` + Expo Go로 빌드 없이 미리보기.
